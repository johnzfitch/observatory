# Phase 6: Better Model Names

## Time: 15 minutes

## File: index.html (or wherever model configs live)

## Problem

Current model names are technical IDs meaningless to users:
- dima806_ai_real
- smogy
- umm_maybe
- prithiv_v2

Users see these during loading and results. No idea what they mean.

## Solution

Create display name mapping:

```js
const MODEL_DISPLAY_NAMES = {
  'dima806_ai_real': 'Style Detector',
  'smogy': 'Texture Analyzer',
  'umm_maybe': 'Face Forensics',
  'prithiv_v2': 'Artifact Scanner'
};

// Helper function
function getModelDisplayName(modelId) {
  return MODEL_DISPLAY_NAMES[modelId] || modelId;
}
```

## Where to Update

### 1. Loading progress
```js
// Before
introAnimation.updateProgress(2, 4);  // "Loading models... 2/4"

// After  
const name = getModelDisplayName(modelName);
updateLoadingText(`Loading ${name}... (2/4)`);
```

### 2. Analysis progress (button)
```js
// Before
btn.textContent = `Analyzing... umm_maybe (3/4)`;

// After
btn.textContent = `${getModelDisplayName('umm_maybe')} (3/4)`;
// Shows: "Face Forensics (3/4)"
```

### 3. Results display
```js
// Before
<div class="model-result">
  <span class="model-name">umm_maybe</span>
  <span class="model-score">87%</span>
</div>

// After
<div class="model-result">
  <span class="model-name">${getModelDisplayName(result.modelId)}</span>
  <span class="model-score">${result.score}%</span>
</div>
```

## Find and Replace

Search codebase for model ID strings and replace with display name calls:

```js
// Search for these patterns:
'dima806_ai_real'
'smogy'
'umm_maybe'
'prithiv_v2'

// In UI-facing code, wrap with:
getModelDisplayName(modelId)
```

Keep raw IDs for:
- Model file paths
- Internal logic
- Console logs (helpful for debugging)
